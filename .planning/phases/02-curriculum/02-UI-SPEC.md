## Interaction Contracts

### Sidebar — Module/Lesson Tree

- Renders a flat tree: Level tabs > Module group headers > Lesson items
- Active lesson: `bg-zinc-800` background, `text-white` label, left border accent `border-l-2 border-white`
- Completed lesson: `text-zinc-400` label + `CheckCircle2` icon (emerald-500) to the right
- Incomplete lesson: `text-zinc-400` label, no icon
- Locked lesson: `text-zinc-600` label + `Lock` icon (zinc-600), cursor-not-allowed, no hover state
- Mobile: sidebar hidden by default, toggled via hamburger button (top-left of lesson page header); renders as overlay drawer

### Level Tabs

- Three tabs: Beginner / Intermediate / Expert
- Active (Beginner): `text-white`, bottom border indicator `border-b-2 border-white`
- Locked (Intermediate, Expert): `text-zinc-600`, `Lock` icon (12px) inline before label, no click interaction, cursor-not-allowed
- Tab height: 40px, label size: 14px / font-normal

### Mark Complete Button

- **Idle state:** Button variant=default (white fill, zinc-950 text), label "Mark complete", no icon
- **Completed state:** Button variant=secondary (zinc-800 fill, white text), label "Completed", `CheckCircle2` icon (emerald-500) inline before label, cursor-default (non-interactive once completed)
- Transition: 150ms ease-in-out on background-color and color
- Positioned: fixed bottom of lesson content area, full-width on mobile / right-aligned on desktop at lesson content width
- One button per lesson page only

### Glossary Tooltip

- **Desktop (hover):** `Tooltip` component from @base-ui/react; triggers on mouse enter of highlighted term; shows one-line definition (max 120 chars) in `bg-zinc-800 text-white text-sm` popover; 8px offset from term; internal padding 8px (sm token)
- **Mobile (click):** Same popover triggered on tap; dismissed on outside tap
- **Term highlighting:** First occurrence only per lesson page; `text-white underline decoration-dotted decoration-zinc-400` inline style; subsequent occurrences plain text

### Glossary Page (/glossary)

- Alphabetical sections with letter headers (`text-2xl font-semibold text-white`, `Separator` below)
- Each term: `text-base font-normal text-white` term label + `text-zinc-400` definition below
- Filter input (top of page): filters terms live (client-side); clears with Button variant=ghost
- Empty filter state: "No matching terms. Try a shorter or different search."

### Locked Level Cards

- `bg-zinc-900` card with `opacity-60` content
- Absolute overlay: `bg-zinc-950/60` covering card
- `Lock` icon (Lucide, 20px, zinc-500) centered in overlay
- No hover state, no cursor-pointer, `aria-disabled="true"`

### Case Study Layout

- Full-width prose: `max-w-3xl mx-auto` centered, `px-6 py-12`
- Same typography as lesson body (16px / 400 / 1.6)
- No sidebar; top breadcrumb nav: `text-sm text-zinc-400` → `text-white` active crumb

## Accessibility

- All interactive sidebar items: `role="link"` or `<a>`, keyboard navigable via Tab
- Locked items: `aria-disabled="true"`, not in tab order (`tabIndex={-1}`)
- Glossary tooltips: `role="tooltip"`, associated via `aria-describedby` on highlighted term
- Mark complete button: `aria-label="Mark lesson complete"` in idle state; `aria-label="Lesson completed"` in completed state
- Lock icons: `aria-hidden="true"` (decorative); text explanation in aria-label of parent
- Color contrast: text-zinc-400 on bg-zinc-950 = 4.8:1 (WCAG AA pass for body text at 16px)
- text-zinc-600 on bg-zinc-900 (locked state): 2.9:1 — acceptable for disabled/locked state (not interactive)
