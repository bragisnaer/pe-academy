The design system is otherwise complete. This is the sole remaining blocker. The ThemeProvider in `app/layout.tsx` currently reads:

```tsx
<ThemeProvider attribute="class" defaultTheme="dark" enableSystem={false} disableTransitionOnChange>
```

next-themes behaviour with `enableSystem={true}`:
- On first visit (no stored pref in localStorage), next-themes reads `prefers-color-scheme` from the OS/browser and applies it.
- `defaultTheme="dark"` acts as the fallback when no system pref is detected (e.g. browsers that do not expose the media query).
- Once a user manually toggles, their choice is stored in localStorage and takes precedence over the system pref on subsequent visits.
- `disableTransitionOnChange` must remain — it prevents the CSS transition flash on theme switch.
</context>
