## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- ThemeToggle button: 44px touch target (height + padding) for WCAG 2.5.5 compliance
- Progress bar track: 8px height (h-2), full width — not subject to spacing scale
- Header height: 56px (h-14) — existing value preserved; not a spacing token

Source: REQUIREMENTS.md DESIGN-04 ("4px-grid spacing"); existing header h-14 from codebase

## ThemeToggle Interaction Contract

The ThemeToggle is the primary new interactive element in Phase 7.

| Property | Value |
|----------|-------|
| Placement | Dashboard header (right side, before sign-out); Lessons layout header (right side) |
| Trigger | Icon-only button, no label |
| Dark icon | `Moon` (lucide) |
| Light icon | `Sun` (lucide) |
| Button size | 36px rendered, 44px touch target (add padding) |
| Button variant | `variant="ghost" size="icon"` (shadcn Button) |
| Mounted guard | Component returns a same-size placeholder (`w-9 h-9`) before mount to prevent hydration mismatch |
| Flash prevention | `next-themes` with `attribute="class"` and `suppressHydrationWarning` on `html` element |
| Default theme | `"dark"` (preserves Bloomberg terminal aesthetic) |
| System pref | Respect `prefers-color-scheme` for first-time visitors with no stored preference |
| Transition | `transition-colors duration-200` on all token-using elements |
| Aria label | `aria-label="Switch to light mode"` / `aria-label="Switch to dark mode"` (dynamic, reflects destination) |

Source: User decision (prompt context) "icon-only sun/moon toggle"; STATE.md "dark-first default", "mounted guard"

## WCAG Compliance Contract

Both themes must pass WCAG 2.1 AA at the end of Phase 7.

| Check | Requirement | Method |
|-------|-------------|--------|
| Text contrast | 4.5:1 minimum (normal text) | Run against light and dark resolved token values |
| Large text contrast | 3:1 minimum (18px+ bold or 24px+ regular) | Checked for h1 (30px) and h2 (20px semibold) |
| Interactive element contrast | 3:1 minimum (button borders, focus rings) | Checked for primary button and ghost button |
| Focus visibility | 2px outline, visible on all themes | `outline-ring/50` already applied via globals.css `@layer base` |
| ThemeToggle touch target | 44px minimum | Enforced via padding on `size="icon"` button |
| Color not sole differentiator | Locked state uses icon + opacity, not color alone | Lock icon (`Lock` lucide) + reduced opacity on locked cards |

Validator: Use Chrome DevTools Accessibility panel or axe-core against both themes before marking DESIGN-05 complete.
