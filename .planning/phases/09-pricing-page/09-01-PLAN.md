## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anonymous → server component | Logged-out users access /pricing; no sensitive data is read or mutated |
| authenticated → server component | Logged-in users access /pricing; only auth.getUser() is called — no user data is displayed or mutated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Information Disclosure | auth.getUser() in PricingPage | accept | Only determines logged-in/out state. No PII, profile data, or subscription status is read or rendered. |
| T-09-02 | Spoofing | Disabled "Upgrade to Pro" button | accept | Button is `disabled` — no form action, no API call. Stripe integration is deferred. No server-side mutation path exists. |
| T-09-03 | Tampering | CTA routing to /signup | accept | /signup is a public route. No CSRF risk on a navigation link. |
| T-09-04 | Denial of Service | Server component rendering | accept | Page is statically renderable (no DB queries beyond auth). CDN-cacheable. Low-value DoS target. |
</threat_model>
