# Milestone v2.0: Public Launch

**Status:** ✅ SHIPPED 2026-04-15
**Phases:** 7–9
**Total Plans:** 3

## Overview

Transformed PE Academy from an internal learning tool into a public-facing product. Added a complete design system with dark/light mode, a marketing homepage at `/`, and a public pricing page — all cleanly separated from the authenticated app via route groups. Stripe checkout deferred to v2.1.

## Phases

### Phase 7: Design System

**Goal**: The app has a complete theme system — light and dark modes work without flash, the html element is no longer hard-coded dark, and all new UI is built on semantic CSS variable tokens rather than raw colour classes.
**Depends on**: Phase 6 (v1.0 complete)
**Plans**: 1 plan (+1 gap closure)

Plans:
- [x] 07-01: Enable system preference detection in ThemeProvider (`enableSystem={true}`)
- [x] 07-gap: Migrate red-* hardcoded colours to destructive semantic tokens (WCAG AA gap closure)

**Details:**
- globals.css defines semantic CSS variable tokens for background, foreground, card, and accent in both light and dark themes
- next-themes ThemeProvider installed; html element no longer has hard-coded dark class; suppressHydrationWarning set
- ThemeToggle component with mounted guard — no flash or layout shift on theme switch
- Finance design language: no decorative gradients, 4px-grid spacing, single accent colour, shadcn Card primitives
- Both themes pass WCAG AA contrast — destructive token migration closed the red-* gap
- `enableSystem={true}` — first-time visitors see theme matching their OS preference
- `defaultTheme="dark"` retained as fallback for browsers that don't expose prefers-color-scheme

**Deferred (non-blocking):** Browser WCAG contrast check on light-mode auth error alerts (text-destructive on bg-destructive/10 — confirm when deployed)

## Milestone Summary

**Key Decisions:**
- Use next-themes with `attribute="class"` to match existing `@custom-variant dark` directive — no CSS strategy change
- `defaultTheme="dark"` preserves Bloomberg terminal aesthetic as deliberate brand signal
- Route groups `app/(marketing)/` + `app/(app)/` for clean URL-transparent separation
- Semantic tokens (bg-background, text-foreground, bg-card) for all new UI; existing app pages keep zinc for v2.0
- Stripe webhook: `await req.text()` not `await req.json()` (documented for v2.1)
- Monthly billing only for v2.0; annual deferred to v2.1
- Pro price: $29/month
- Free tier: Level 1; Pro: Levels 2 & 3

**Issues Resolved:**
- First-time visitors were stuck in dark mode regardless of OS preference (fixed: enableSystem={true})
- Hardcoded red-* colour classes failed WCAG AA in light mode (fixed: destructive token migration)
- Anon users couldn't see news teaser on homepage (fixed: RLS policy for anon reads)

**Issues Deferred to v2.1:**
- Stripe checkout integration (subscription_tier column, webhook handler, Stripe API routes)
- Annual billing toggle
- Legal pages (Terms of Service, Privacy Policy)
- OG card preview confirmation (needs deployed URL)
- WCAG contrast browser check on light-mode auth error alerts

**Technical Debt Incurred:**
- `next build` Turbopack config warning (pre-existing, unrelated to v2.0 — next.config missing turbopack key)
- Stripe CTA is a placeholder disabled button — must be replaced in v2.1 before marketing push
